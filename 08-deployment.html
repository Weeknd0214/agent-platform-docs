    🐳 部署与运维

    Agent Platform 支持开发模式（直接本地运行）和生产模式（Docker Compose 容器化）两种部署方式， Nginx 反向代理统一对外入口，支持 HTTPS 和域名绑定。

    开发环境部署

    一键启动脚本

    bash 
    # Linux/macOS
./scripts/start-dev.sh

# Windows (PowerShell)
.\scripts\start-dev.bat

    脚本自动完成以下操作：

    • 自动检测内网 IP 并写入 .env.local
    • 检查 langgraph、npm 依赖
    • 后台启动 LangGraph Agent 服务（端口 2024）
    • 安装前端依赖（如未安装）并启动 Next.js 开发服务器（端口 3000）
    • Ctrl+C 优雅停止所有进程

    手动分步启动

    bash 
    # 终端 1：启动后端 Agent
cd backend
pip install -r requirements.txt
langgraph dev --host 0.0.0.0 --port 2024

# 终端 2：启动前端
cd frontend/agent-chat-ui
npm install
npm run dev

    容器化部署

    Backend Dockerfile

    dockerfile 
    FROM python:3.12-slim-bookworm

WORKDIR /app

# 安装系统依赖（pygraphviz 需要 graphviz 开发头文件）
RUN apt-get update && apt-get install -y --no-install-recommends \
    graphviz libgraphviz-dev pkg-config gcc python3-dev \
    && rm -rf /var/lib/apt/lists/*

# 分批安装依赖（避免 pip 依赖解析深度超限）
RUN pip install --no-cache-dir \
    "langgraph-cli[inmem]" langgraph langchain \
    langchain-core langchain-ollama httpx

# 安装 CV/数据处理依赖
ENV GRAPHVIZ_DIR=/usr
RUN pip install --no-cache-dir \
    pillow numpy opencv-python-headless \
    tqdm requests rarfile py7zr pyyaml \
    moviepy lxml typing-extensions \
 && pip install --no-cache-dir pygraphviz

COPY . /app/
EXPOSE 2024
CMD ["langgraph", "dev", "--host", "0.0.0.0", "--port", "2024"]

    Docker Compose 启动

    bash 
    # 首次启动（构建镜像）
docker compose up -d --build

# 查看所有服务状态
docker compose ps

# 查看特定服务日志
docker compose logs -f backend
docker compose logs -f rag

# 重启某个服务
docker compose restart backend

# 停止所有服务
docker compose down

# 停止并清除数据卷（危险！会删除数据库数据）
docker compose down -v

    Nginx 反向代理

    安装配置

    bash 
    # 复制配置文件
sudo cp deploy/nginx/agent-platform.conf /etc/nginx/sites-available/
sudo ln -s /etc/nginx/sites-available/agent-platform.conf /etc/nginx/sites-enabled/

# 修改域名
sudo nano /etc/nginx/sites-available/agent-platform.conf
# 将 server_name your-domain.com; 改为实际域名或 IP

# 测试并重载配置
sudo nginx -t
sudo nginx -s reload

    核心配置说明

    nginx 
    server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 50M;  # 支持大文件上传

    location / {
        proxy_pass http://127.0.0.1:3000;  # Next.js 前端
        proxy_http_version 1.1;
        # WebSocket 升级支持
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        # SSE 长连接不超时
        proxy_read_timeout 86400;
    }
}

    启用 HTTPS（Let's Encrypt）

    bash 
    # 安装 certbot
sudo apt install certbot python3-certbot-nginx

# 申请证书（自动修改 Nginx 配置）
sudo certbot --nginx -d your-domain.com

# 自动续期（已由 certbot 添加到 cron）
sudo certbot renew --dry-run

    环境变量配置

    后端关键变量

    变量名默认值说明
    OLLAMA_BASE_URLhttp://localhost:11434Ollama 服务地址
    OLLAMA_MODELqwen2.5:7b使用的模型名称
    MOTOVIS_RAG_URLhttp://rag:9380RAG 服务（Docker 内网）
    SAFETY_QA_DIALOG_ID—商用车 QA 默认 Dialog ID

    前端关键变量

    变量名默认值说明
    NEXT_PUBLIC_AGENT_API_URLhttp://localhost:2024LangGraph Backend
    NEXT_PUBLIC_RAG_URLhttp://localhost:9380RAG 服务地址
    DATABASE_URL—MySQL 连接串
    SESSION_SECRET—Session 加密密钥（生产必填）

    监控运维

    前端监控仪表盘

    管理员登录后可访问"全局监控"视图（MonitoringDashboard）， 展示：

    • 各 Agent 服务健康状态
    • API 请求量统计（网关 API 接口）
    • 资源使用情况（ResourceMonitorView）：CPU/内存/GPU 占用
    • 网关流量分析（GatewayPlanModule）

    日志查看

    bash 
    # Docker 服务日志
docker compose logs -f --tail=100 backend
docker compose logs -f --tail=100 rag

# 开发模式日志
tail -f /tmp/agent.log
tail -f /tmp/frontend.log

# Nginx 访问日志
tail -f /var/log/nginx/agent-platform-access.log

    健康检查

    bash 
    # Backend Agent
curl http://localhost:2024/health

# RAG 服务
curl http://localhost:9380/v1/system/status

# MySQL
docker exec agent-platform-mysql mysqladmin ping -uroot -pinfini_rag_flow

# Redis
docker exec agent-platform-redis redis-cli -a infini_rag_flow ping

# Elasticsearch
curl http://localhost:9200/_cluster/health
