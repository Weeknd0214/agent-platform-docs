    🤖 算法处理系统

    算法处理系统是 Agent Platform 的核心业务价值所在， 以 FCOS（Fully Convolutional One-Stage Object Detection）为基础， 面向后装市场的 DMS/BSD/HOD 等多个安全驾驶场景提供自动化训练、评估和模型转换能力。

    FCOS 算法实现

    算法概述

    FCOS 是一种 Anchor-Free 的单阶段目标检测算法，相比传统 Anchor-Based 方法， 无需预设锚框，通过逐像素预测目标的中心点、边界框偏移和置信度来实现检测。

    网络架构

    输入图像
    H xd7 W xd7 3 RGB 张量
    Backbone（RepVGGDet）
    多阶段特征提取 — 训练时多分支并行 / 部署时等效单分支（推理加速）
    FPN（Feature Pyramid Network）
    P3 / P4 / P5 多尺度特征融合：小目标用浅层，大目标用深层
    FCOSHead
    分类分支目标类别预测（Focal Loss）
    回归分支l/t/r/b 边界框偏移（IoU Loss）
    中心度分支Center-ness 权重（BCE Loss）
    后处理（NMS）
    非极大値抑制 → 输出最终检测框

    RepVGG 结构重参数化

    Backbone 采用 RepVGG 设计，在训练阶段使用多分支结构（3×3 + 1×1 + Identity）， 在部署阶段通过结构重参数化（Structural Re-parameterization）将多分支等效合并为 单个 3×3 卷积，实现推理速度提升而不损失精度。

    架构对比 
    # 训练时（多分支）：
output = conv3x3(x) + conv1x1(x) + x    # 三路并行

# 推理时（单分支，等效合并后）：
output = conv3x3_merged(x)               # 推理快 ~1.5x

    ℹ️

    _fudai 变体

    带 _fudai 后缀的目录（如 dms_eye_det_fudai） 是针对"副代"（外挂/违规装置遮挡摄像头场景）的专项适配版本， 增强了对遮挡和异常安装角度的鲁棒性。

    BSD（盲区监测系统）

    BSD 模块专注于车辆盲区目标检测，主要检测： 摩托车、自行车、行人、小轿车在车辆侧后方盲区的出现。 支持不同摄像头安装位置（左盲区/右盲区/后方）的参数适配。

    HOD（车内危险目标检测）

    HOD（Hazardous Object Detection）包含 11 个子模块，检测车厢/驾驶舱内的危险物品：

    刀具 枪支 危险品包装 电子烟 液体容器 以及其他合规检测项目

    训练流程管理

    任务处理流程

    后装算法迭代 Agent 通过 LangGraph 的 task_proc 子图管理完整的训练流程：

    1. 任务解析：LLM 从用户自然语言中提取任务类型（DMS2.0）、 功能（人眼）、任务名称、处理类型（train/eval/model_cvt）
    2. 数据下载（可选）：从 HTTP 服务器拉取标注数据集 → 触发 interrupt() 等待用户确认
    3. 数据标注（训练任务）：将原始数据转换为 FCOS 训练格式， 生成 annotation JSON 文件
    4. 训练/预测/转换：调用对应场景目录下的训练脚本， 支持 Shell 调用和 Python API 两种方式
    5. 结果汇报：训练指标（mAP、Loss 曲线）回传到对话界面

    推理优化技术

    模型转换

    通过 model_cvt（模型转换）任务，将 PyTorch 训练模型转换为 目标芯片的部署格式：

    目标平台转换格式说明
    通用平台ONNX标准中间表示，便于跨平台部署
    NT98323（DMS2.0）厂商专用格式星芯通芯片优化格式
    241/216 平台平台特化格式量化 + 算子融合优化

    量化优化

    • INT8 量化：减少模型体积 75%，推理速度提升 2-4x
    • 算子融合：Conv + BN + ReLU 等连续算子融合为单算子
    • RepVGG 重参数化：训练多分支 → 部署单分支（无精度损失）

    模型评估体系

    评估模块（eval 任务类型）提供完整的检测性能评估：

    评估指标说明
    mAP各类别平均精度均值（IoU=0.5）
    mAP@[.5:.95]多 IoU 阈值下的 mAP，更严格的评估
    FPS目标硬件上的推理帧率
    误报率/漏报率针对业务场景的自定义评估维度
