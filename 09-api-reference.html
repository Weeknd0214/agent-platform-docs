    📡 API 参考

    Agent Platform 提供四套 API：LangGraph 工作流 API、前端认证 API、 RAG 知识库 API 和网关统计 API，全部通过 Next.js API Routes 代理访问。

    LangGraph API

    LangGraph Server 兼容 OpenAI Assistants API 风格， 完整文档见 LangGraph Cloud API Reference。

    Thread（对话线程）

    POST /api/langgraph/threads

    创建新的对话线程。

    GET /api/langgraph/threads/{thread_id}

    获取线程信息与最新状态（消息历史）。

    POST /api/langgraph/threads/{thread_id}/runs/stream

    向 Agent 发送消息并以 SSE 流式方式接收回复。

    请求体 
    {
  "assistant_id": "agent",
  "input": {
    "messages": [
      { "role": "human", "content": "帮我训练 DMS2.0 任务" }
    ]
  },
  "stream_mode": ["values", "messages"],
  "config": {
    "configurable": {
      "thread_id": "thread-uuid-xxx",
      "context": {
        "knowledge_memory": false
      }
    }
  }
}
    SSE 响应事件 
    // 消息流事件
event: messages/partial
data: {"content": "正在分析", "type": "AIMessageChunk"}

// 完整状态更新
event: values
data: {"messages": [...], "task_name": "DMS2.0-人眼-test"}

// 人在回路中断
event: interrupt
data: [{"action_request": {"action": "confirm_download"}, "description": "确认下载数据？"}]

    人在回路恢复

    POST /api/langgraph/threads/{thread_id}/runs/stream
    恢复执行 
    {
  "assistant_id": "agent",
  "command": {
    "resume": { "action": "confirm", "args": {} }
  }
}

    Assistants（Graph 注册表）

    GET /api/langgraph/assistants

    获取所有注册的 Graph（即 langgraph.json 中配置的 graphs）。

    认证 API

    登录

    POST /api/auth/login
    请求/响应 
    // 请求
{ "username": "admin", "password": "xxx", "rememberMe": true }

// 响应（成功）
{
  "success": true,
  "user": {
    "username": "admin",
    "userRole": 1,
    "isAdmin": true
  },
  "userConfig": {
    "langgraphApiUrl": "http://localhost:2024",
    "assistantId": "agent"
  }
}

// 响应（失败）
{ "success": false, "message": "用户名或密码错误" }

    获取 Session

    GET /api/auth/session

    验证当前 cookie 是否有效，返回用户信息。

    登出

    POST /api/auth/logout

    清除 Session cookie，返回 { success: true }。

    RAG API

    RAG 服务基础路径：http://localhost:9380/v1， 通过前端代理路径 /api/rag 访问。 所有接口需要携带 Authorization Bearer Token。

    知识库管理

    POST /v1/kb/create

    创建知识库（Knowledge Base）。

    GET /v1/kb/list

    获取知识库列表。

    文档管理

    POST /v1/document/upload

    上传文档到指定知识库（multipart/form-data）。

    POST /v1/document/run

    触发文档解析（异步）。

    json 
    // POST /v1/document/run
{ "doc_ids": ["doc-id-1", "doc-id-2"] }

    对话应用

    POST /v1/dialog/create

    创建对话应用（Dialog），关联知识库和 LLM 配置。

    POST /v1/api/completion

    对话问答接口（商用车 QA Agent 调用此接口）。

    请求/响应 
    // 请求
{
  "dialog_id": "xxx",
  "messages": [
    { "role": "user", "content": "DMS2.0 的安全标准是什么？" }
  ],
  "stream": false
}

// 响应
{
  "data": {
    "answer": "根据文档，DMS2.0 的安全标准...",
    "reference": {
      "chunks": [
        { "content": "相关段落...", "doc_name": "安全规范.pdf", "positions": [] }
      ]
    }
  }
}

    记忆管理 API

    POST /v1/memory/create
    GET /v1/memory/list
    DELETE /v1/memory/{memory_id}

    网关 API

    前端"全局监控"模块通过网关 API 获取流量统计和资源使用数据。

    GET /api/gateway-stats

    获取 API 网关流量统计（请求量、响应时间、错误率）。

    GET /api/resource-monitor

    获取服务器资源使用情况（CPU、内存、GPU 利用率）。

    管理 API

    GET /api/admin/users

    获取所有用户列表（需管理员权限）。

    POST /api/admin/users

    创建新用户。

    PUT /api/admin/users/{id}

    更新用户信息（角色、密码、启用状态）。

    DELETE /api/admin/users/{id}

    删除用户（级联删除配置）。
