    👥 用户管理系统

    Agent Platform 内置完整的用户认证与权限管理系统，基于 MySQL 存储用户数据， 支持管理员/普通用户角色划分，并允许每个用户绑定独立的 LangGraph 配置。

    数据库设计

    用户表 — users

    sql 
    CREATE TABLE `users` (
  `id`            INT AUTO_INCREMENT PRIMARY KEY,
  `username`      VARCHAR(50)  NOT NULL UNIQUE COMMENT '用户名',
  `password_hash` VARCHAR(255) NOT NULL COMMENT 'bcrypt 哈希',
  `user_role`     TINYINT(1)  NOT NULL DEFAULT 1
                  COMMENT '1=算法/工程 2=产品/测试 3=运维',
  `is_admin`      TINYINT(1)  DEFAULT 0 COMMENT '超级管理员',
  `avatar_url`    VARCHAR(500) DEFAULT '',
  `created_at`    TIMESTAMP   DEFAULT CURRENT_TIMESTAMP,
  `updated_at`    TIMESTAMP   DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `last_login`    TIMESTAMP   NULL,
  `is_active`     TINYINT(1)  DEFAULT 1,
  INDEX `idx_username`  (`username`),
  INDEX `idx_user_role` (`user_role`),
  INDEX `idx_is_admin`  (`is_admin`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

    用户配置表 — user_configs

    sql 
    CREATE TABLE `user_configs` (
  `id`                 INT AUTO_INCREMENT PRIMARY KEY,
  `user_id`            INT          NOT NULL,
  `config_name`        VARCHAR(100) NOT NULL DEFAULT 'default',
  `langsmith_api_key`  VARCHAR(255) DEFAULT '',
  `langgraph_api_url`  VARCHAR(255) DEFAULT 'http://localhost:2024',
  `assistant_id`       VARCHAR(50)  DEFAULT 'agent',
  `is_default`         TINYINT(1)   DEFAULT 0,
  `created_at`         TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_user_config` (`user_id`, `config_name`),
  FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

    用户配置表支持每个用户绑定多套 LangGraph 配置（config_name 区分）， 例如连接不同的 Backend 实例或不同的 Assistant ID。

    角色权限

    角色user_roleis_admin可访问功能
    超级管理员 任意 1 所有功能 + 全局监控 + 权限管理（增删用户）
    算法/工程 1 0 Agent 对话、运行时、RAG 访问
    产品/测试 2 0 Agent 对话、Marketplace 浏览
    运维 3 0 Agent 对话、资源监控

    认证流程

    登录

    1. 前端 POST /api/auth/login，携带 username, password, rememberMe
    2. API Route 查询 MySQL，bcrypt.compare 验证密码
    3. 验证通过，生成 Session：设置 HttpOnly cookie（rememberMe=true 时有效期 30 天）
    4. 返回 { success: true, user: {...}, userConfig: {...} }
    5. AuthProvider 更新全局状态，路由跳转到主页

    Session 验证

    typescript 
    // 每次页面加载时自动验证
const checkAuth = async () => {
  const res = await fetch('/api/auth/session');
  const data = await res.json();
  if (data.success && data.user) {
    setUser(data.user);
    setUserConfig(data.userConfig);
  }
};

    登出

    typescript 
    const logout = async () => {
  await fetch('/api/auth/logout', { method: 'POST' });
  setUser(null);
  setUserConfig(null);
  router.push('/login');
};

    管理后台

    超级管理员在侧边栏可访问"权限管理"入口，进入 AdminView 组件， 提供以下功能：

    • 用户列表：查看所有用户，支持搜索过滤
    • 新建用户：填写用户名、密码、角色，创建后自动哈希密码存入 MySQL
    • 编辑用户：修改角色、重置密码、启用/禁用账号
    • 删除用户：级联删除用户配置（ON DELETE CASCADE）
    • 用户配置管理：为每个用户设置独立的 LangGraph API URL

    安全提示

    生产环境部署后，请立即修改默认管理员密码（默认 motovis）。 密码通过 bcrypt 哈希存储，强度系数 10 轮，无法反推原始密码。

    密码策略

    使用 bcrypt（cost factor = 10）哈希存储， 通过 lib/auth/password-hash.ts 提供统一的哈希与验证接口：

    typescript 
    import bcrypt from 'bcryptjs';

export const hashPassword = (password: string): string => {
  return bcrypt.hashSync(password, 10);
};

export const verifyPassword = (password: string, hash: string): boolean => {
  return bcrypt.compareSync(password, hash);
};
