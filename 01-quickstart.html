    🚀 快速开始

    按照本指南，10 分钟内在本地完整启动 Agent Platform， 包括前端、后端、RAG 服务及所有依赖基础设施。

    环境要求

    依赖版本要求用途
    Docker24.0+容器化运行基础设施
    Docker Compose2.20+多服务编排
    Node.js18.0+前端开发服务器
    Python3.11+后端 Agent 服务
    Ollama最新本地 LLM 运行时
    ✅

    Windows 用户

    推荐使用 WSL2 + Docker Desktop，确保 Docker Compose v2 可用（docker compose 而非 docker-compose）。

    第一步：克隆项目

    bash 
    git clone https://github.com/motovis/agent-platform.git
cd agent-platform

    第二步：配置环境变量

    bash 
    # 根目录环境变量（Docker Compose 使用）
cp .env.example .env

# RAG 服务环境变量
cp rag/.env.example rag/.env

# 后端环境变量
cp backend/.env.example backend/.env

# 前端环境变量
cp frontend/agent-chat-ui/.env.example frontend/agent-chat-ui/.env

    关键环境变量说明

    变量名默认值说明
    NEXT_PUBLIC_AGENT_API_URLhttp://localhost:2024LangGraph 后端地址
    NEXT_PUBLIC_RAG_URLhttp://localhost:9380RAG 服务地址
    OLLAMA_MODELqwen2.5:7bOllama 使用的模型
    OLLAMA_BASE_URLhttp://localhost:11434Ollama 服务地址
    MOTOVIS_RAG_URLhttp://rag:9380后端访问 RAG 的地址（Docker 内网）

    第三步：启动基础设施 + RAG 服务

    bash 
    # 启动 MySQL、Redis、MinIO、Elasticsearch、RAG、Backend
docker compose up -d --build

# 查看启动状态
docker compose ps

    首次启动会拉取镜像并构建，约需 5-10 分钟。成功后各服务端口如下：

    服务端口说明
    MySQL3307数据库（映射到宿主机 3307）
    Redis6379缓存
    MinIO9000对象存储
    Elasticsearch9200向量/全文检索
    RAG 服务9380RAGFlow REST API
    Backend Agent2024LangGraph Server

    第四步：启动前端

    bash 
    cd frontend/agent-chat-ui

# 安装依赖（首次）
npm install

# 初始化数据库（MySQL 用户表等）
node scripts/init-users.js

# 启动开发服务器
npm run dev

    前端默认运行在 3000 端口，打开浏览器访问 http://localhost:3000。

    ℹ️

    默认账号

    运行 init-users.js 后，默认管理员账号为 admin / admin123，请登录后立即修改密码。

    第五步：配置 Ollama 模型

    bash 
    # 安装 Ollama（macOS/Linux）
curl -fsSL https://ollama.com/install.sh | sh

# 拉取推荐模型（qwen2.5:7b）
ollama pull qwen2.5:7b

# 或者使用其他支持工具调用的模型
ollama pull qwen2.5:14b
ollama pull llama3.1:8b

# 验证模型可用
ollama list

    第六步：验证部署

    1. 访问前端：打开 http://localhost:3000，用 admin 账号登录， 应看到 Agent Marketplace 页面。
    2. 测试后装 Agent：在 Marketplace 选择"后装算法迭代 Agent"， 发送"我需要训练 DMS2.0 任务"，观察 Agent 是否返回结构化回复。
    3. 验证 RAG：选择"商用车主动安全 Agent"，开启"知识库记忆"， 上传安全规范文档，提问验证检索效果。
    4. 检查健康状态：访问 http://localhost:2024/health（Backend） 和 http://localhost:9380/v1/system/status（RAG）。
    ⚠️

    常见问题

    如果 Elasticsearch 启动失败，通常是因为 vm.max_map_count 设置过低。 Linux 下执行 sudo sysctl -w vm.max_map_count=262144， Windows WSL2 下在 /etc/sysctl.conf 中添加该配置。

    开发模式

    如果你想单独开发后端，可以跳过 Docker 构建，直接本地运行：

    bash 
    cd backend

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动 LangGraph 开发服务器
langgraph dev --config langgraph.json
