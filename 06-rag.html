    📚 RAG 功能模块

    RAG（Retrieval-Augmented Generation）模块基于 RAGFlow 构建， 为商用车主动安全 Agent 提供知识库检索增强能力， 支持多数据源接入、文档解析、向量检索和对话问答。

    文档处理系统

    支持的文档格式

    PDF Word (.docx/.doc) Excel (.xlsx/.xls) PowerPoint (.pptx) Markdown HTML TXT CSV 图片（OCR）

    文档处理流程

    1. 上传：通过 API 上传文件到 MinIO 对象存储
    2. 解析：DeepDoc 解析器提取文本（PDF 含版面分析、表格识别、图片 OCR）
    3. 切片：按语义或固定大小切分文档片段（chunk）
    4. 嵌入：LLM Embedding 模型生成向量表示
    5. 索引：写入 Elasticsearch（向量索引 + 倒排索引双路）

    DeepDoc 视觉分析

    deepdoc/vision/ 包含专门针对文档版面分析的模型：

    • 版面检测：识别页面中的标题、段落、表格、图片区域
    • 表格识别：提取表格结构，转换为结构化数据
    • OCR：图片中的文字识别
    • 公式识别：数学公式的文本化

    检索增强系统

    检索策略

    RAG 支持多种检索策略组合：

    策略说明适用场景
    向量检索Embedding 语义相似度匹配语义查询，语义近似但字面不同
    全文检索BM25 关键词匹配精确术语查询
    混合检索向量 + 全文 加权融合（RRF）通用最佳实践
    知识图谱GraphRAG 实体关系检索多跳推理场景

    对话问答流程

    ① 用户问题
    自然语言输入
    ② Query 理解
    意图分析 + 关键词提取 + 向量嵌入
    ③ 混合检索（Elasticsearch）
    kNN 向量检索 | BM25 全文检索
    ④ 重排序（Cross-Encoder Reranking）
    精准排序，取 Top-K 相关片段
    ⑤ 上下文构建
    相关文档片段 + 引用来源元数据
    ⑥ LLM 生成（附引用标注）
    Ollama qwen2.5:7b 生成含引用的回答
    ⑦ 返回答案 + 参考文献
    含引用片段和文档来源

    内存管理系统

    rag/memory/ 提供对话记忆管理功能，支持跨会话的长期记忆存储和检索：

    • 短期记忆：当前对话上下文窗口内的消息历史
    • 长期记忆：提取关键信息（实体、偏好）存储到向量数据库，跨会话检索
    • 记忆服务：memory/services/ 提供记忆的增删改查 API
