    数据流架构

    对话请求数据流

    ① 用户发送消息
    前端 useStream Hook 触发请求
    ② Next.js API Route 代理
    POST /api/langgraph/threads/{'{id}'}/runs/stream → 转发到 LangGraph Server
    ③ LangGraph Backend :2024
    graph.invoke / graph.stream — 执行 Agent 状态图节点
    🔵 直连模式Ollama LLM :11434 → 返回 AI 回复
    🟢 RAG 模式RAG Service :9380 → ES 向量检索 → LLM 生成含引用答案
    ④ SSE 流式响应回传前端
    event: messages/partial → React useStream 实时渲染消息

    人在回路（Human-in-the-Loop）数据流

    ① node_task_proc_assistant
    LLM 解析任务类型 / 名称 / 参数，路由到 node_task_proc
    ② 下载数据步骤 → interrupt()
    图执行暂停，前端收到 HumanInterrupt 事件，显示"确认继续"按钮
    ③ 用户点击确认
    POST .../runs/stream { command: { resume: ... } } → 图从中断点恢复
    ④ 执行训练 / 预测 → 返回结果
    task_proc 子图继续执行，完成后回传训练指标

    部署架构

    Docker Compose 编排关系

    yaml 
    services:
  mysql:      # 基础数据库，最先启动
  redis:      # 缓存服务
  minio:      # 对象存储
  elasticsearch:  # 向量搜索引擎
  rag:        # depends_on: mysql, redis, minio, elasticsearch
  backend:    # depends_on: rag
              # environment: MOTOVIS_RAG_URL=http://rag:9380

    服务间通信

    Docker 内部网络中，Backend 通过 http://rag:9380 访问 RAG 服务 （容器名作为 hostname），宿主机访问使用 http://localhost:9380。

    安全架构

    认证授权

    • Session 认证：Next.js API Route 生成 HttpOnly cookie，客户端无法直接读取 token
    • 角色控制：isAdmin 字段区分管理员与普通用户，管理员可访问全局监控和权限管理
    • RAG API Token：RAG 服务使用 Bearer Token 认证，前端通过 Next.js API Route 代理，隐藏 token

    数据安全

    • 所有密码字段经过 bcrypt 哈希存储
    • Elasticsearch 启用 X-Pack 安全认证（xpack.security.enabled=true）
    • 生产环境建议启用 Nginx HTTPS（Let's Encrypt）
    • Docker 服务间通信默认不暴露到宿主机网络

    SSE 连接安全

    Nginx 配置 proxy_read_timeout 86400，保持 SSE 长连接不超时； proxy_set_header Upgrade $http_upgrade 支持 WebSocket 升级。
