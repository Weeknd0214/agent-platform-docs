    ⚙️ 后端服务

    后端基于 LangGraph Server 构建，提供两个独立的 AI Agent 工作流： 后装算法迭代 Agent（houzhuang_agent）和商用车主动安全 QA Agent（safety_qa_agent）， 通过统一的 LangGraph API 对外提供服务。

    LangGraph 智能体架构

    什么是 LangGraph

    LangGraph 是 LangChain 出品的状态图（StateGraph）框架，将 AI Agent 的执行流程 建模为有向图：节点（Node）执行具体逻辑，边（Edge）定义路由条件， 状态（State）在节点间传递和共享。

    ℹ️

    LangGraph Server

    LangGraph Server 自动处理 checkpoint（持久化）和 streaming， 无需手动实现 MemorySaver，通过 langgraph.json 声明 graph 入口。

    服务注册 — langgraph.json

    json 
    {
  "graphs": {
    "agent": "./houzhuang_agent/task_proc_agent_v2.py:app",
    "safety_qa_agent": "./safety_qa_agent/safety_qa_agent.py:app"
  },
  "python_version": "3.11",
  "dependencies": ["."]
}

    后装算法迭代 Agent

    状态定义

    python 
    class State(TypedDict, total=False):
    messages: Annotated[list[BaseMessage], add_messages]  # 对话消息（reducer）
    user_input: str          # 原始用户输入
    task_name: str           # 任务名称
    task_proc_type: str      # 任务类型：train/eval/model_cvt
    model_tag: str           # 模型标识：时间戳或 latest
    data_need_download: bool # 是否需要下载数据
    data_need_labeling: bool # 是否需要标注（训练任务）
    task_proc_output: str    # 任务处理输出
    human_interrupt: list    # 人在回路中断数据
    user_id: str
    thread_id: str
    task_type: str           # DMS1.0/DMS2.0/BSD/HOD
    task_func: str           # 人眼/人头 等功能

    图结构

    __start__
    图入口
    node_task_proc_assistant
    LLM 解析用户意图，返回 AssistantResponseFormat
    → node_end任务完成，直接结束
    → node_user_input需要更多信息，等待前端下一轮消息
    → node_task_proc task_proc 子图：下载 → interrupt() → 确认 → 训练/预测 → 结果

    助手节点 — node_task_proc_assistant

    使用 create_agent 创建带结构化输出的 LLM Agent， 通过 AssistantResponseFormat 约束输出格式：

    python 
    @dataclass
class AssistantResponseFormat:
    response: str = ""          # 助手回答文本
    task_proc_type: str = ""    # train / eval / model_cvt
    task_name: str = ""         # 完整任务名称
    data_need_download: bool = True
    data_need_labeling: bool = True
    model_tag: str = "latest"
    next_node: str = ""         # 路由目标节点名称

    双模式响应

    _rag_chat_node 根据 context.knowledge_memory 字段 决定使用哪种响应方式：

    模式条件实现
    直连模式 knowledge_memory = false 直接调用 Ollama LLM，与后装 Agent 共用 ollama_config
    RAG 模式 knowledge_memory = true 调用 MotovisRAG chat_completion，返回含引用来源答案

    任务处理系统

    任务识别与分类

    LLM Prompt 明确定义了任务分类规则：

    • 识别 DMS1.0/DMS2.0/BSD/HOD 任务类型
    • 识别处理类型：训练（train）/ 预测（eval）/ 模型转换（model_cvt）
    • 提取任务名称（格式：任务类型-功能-信息串）
    • 判断是否需要数据下载/标注

    LLM 集成

    Ollama 配置 — ollama_config.py

    python 
    # backend/ollama_config.py
from langchain_ollama import ChatOllama

def get_chat_ollama():
    return ChatOllama(
        base_url=os.getenv("OLLAMA_BASE_URL", "http://localhost:11434"),
        model=os.getenv("OLLAMA_MODEL", "qwen2.5:7b"),
        temperature=float(os.getenv("OLLAMA_TEMPERATURE", "0.7")),
    )
    ✅

    模型选择建议

    后装 Agent 需要结构化输出能力，推荐使用 qwen2.5:14b 或更大模型； 安全 QA 直连模式下 qwen2.5:7b 即可满足需求。

    流式响应处理

    LangGraph Server 默认支持 SSE 流式输出，前端通过以下方式订阅：

    typescript 
    // 前端请求流式对话
POST /threads/{thread_id}/runs/stream
Content-Type: application/json

{
  "assistant_id": "agent",
  "input": { "messages": [{ "role": "human", "content": "..." }] },
  "stream_mode": ["values", "messages"],
  "config": {
    "configurable": {
      "thread_id": "xxx",
      "context": { "knowledge_memory": false }
    }
  }
}

    响应为 SSE 事件流，包含 data、metadata、interrupt 等事件类型，useStream hook 自动处理并更新 React 状态。
