    🖥️ 前端应用

    组件架构

    主应用入口 — agent-platform/index.tsx

    根组件负责视图路由（marketplace | runtime | monitoring | admin）， 管理当前选中的 Agent、选中的视图 Tab，以及移动端侧边栏的开关状态。

    typescript 
    // 视图切换模型
type View = "marketplace" | "runtime" | "monitoring" | "admin";

// Marketplace → 选择 Agent → Runtime（含 Chat/Prompt/RAG 等 Tab）
const [currentView, setCurrentView] = useState<View>("marketplace");
const [currentAgent, setCurrentAgent] = useState<string | null>(null);

    侧边栏 — Sidebar.tsx

    根据当前 Agent 类型动态展示不同的菜单项。商用车主动安全 QA 仅显示"对话交互"和"访问 RAG"， 后装算法迭代 Agent 则展示完整的 Prompt/Models/Skills/Memory/Audit 等菜单。

    Marketplace — MarketplaceView.tsx

    展示所有可用的 Agent 卡片（来自 config/agents.ts 中的 AGENT_CONFIGS）， 用户点击"进入运行时"按钮后，切换到 Runtime 视图并将选中的 Agent 配置注入 Stream Provider。

    对话界面 — thread/index.tsx

    核心对话组件，负责：

    • 渲染消息历史（AI 回复 Markdown 渲染 + 代码高亮）
    • 输入框（文字 + 多模态文件上传）
    • 人在回路中断 UI（agent-inbox/ 子组件）
    • 流式消息动画

    状态管理架构

    Stream Provider — 流式通信核心

    providers/Stream.tsx 是整个应用最核心的状态管理层，封装了 @langchain/langgraph-sdk/react 的 useStream hook。

    typescript 
    // Stream 状态类型定义
type StateType = {
  messages: Message[];
  ui?: UIMessage[];
};

// 支持的中断类型（人在回路）
type InterruptType = HumanInterruptValue[] | HumanInterruptValue;

// HumanInterrupt 结构（与后端 task_proc 子图一致）
interface HumanInterruptValue {
  action_request?: { action: string; args?: Record<string, unknown> };
  config?: {
    allow_accept?: boolean;
    allow_ignore?: boolean;
    allow_respond?: boolean;
    allow_edit?: boolean;
  };
  description?: string;
}

    Stream Provider 的关键逻辑：

    • URL 规范化：自动补全 http:// 前缀，将 0.0.0.0 替换为 localhost
    • configurable 注入：商用车 QA Agent 自动注入 motovis_dialog_id 到请求参数
    • thread 管理：通过 URL query 参数（nuqs）持久化 thread_id，支持对话历史恢复
    • context 传递：knowledge_memory 开关状态随每轮消息传入后端

    Auth Provider — 认证状态

    typescript 
    interface AuthContextType {
  user: User | null;
  userConfig: UserConfig | null;
  isLoading: boolean;
  isAuthenticated: boolean;
  login: (username: string, password: string, rememberMe: boolean) => Promise<boolean>;
  logout: () => Promise<void>;
}

    checkAuth() 在组件挂载时调用 GET /api/auth/session 验证 cookie， 实现页面刷新后的自动登录恢复。

    Dialog-Thread 状态持久化

    商用车 QA Agent 中，每个 LangGraph thread 需要绑定一个 MotovisRAG Dialog ID。 lib/motovis-safety-dialog.ts 使用 localStorage 维护 threadId → dialogId 的映射表，防止多会话时 Dialog 错乱。

    API 集成

    API Route 代理层

    所有后端调用均通过 Next.js API Routes 代理，避免前端直接暴露后端地址和 token：

    API 路径代理到功能
    /api/auth/*MySQL（直连）登录/登出/Session
    /api/langgraph/*LangGraph :2024Agent 对话/流
    /api/rag/*RAG :9380知识库操作
    /api/ollama/*Ollama :11434模型列表
    /api/training/*Backend :2024训练任务管理
    /api/admin/*MySQL（直连）用户管理

    LangGraph SDK 集成

    多模态支持

    hooks/use-file-upload.tsx 处理文件上传， lib/multimodal-utils.ts 负责格式转换：

    • 图片：转为 data:image/jpeg;base64,...，封装为 image_url 块
    • PDF：直连 Ollama 模式下提示用户补充文字说明；RAG 模式下通过知识库解析
    • 预览：MultimodalPreview.tsx 在消息气泡中展示图片缩略图

    认证系统

    登录流程

    1. 用户提交登录表单 → POST /api/auth/login
    2. API Route 查询 MySQL 用户表，验证 bcrypt 密码
    3. 验证通过 → 创建 HttpOnly Session Cookie
    4. 前端 AuthProvider 更新 user 状态，路由跳转到主页

    路由守卫

    app/page.tsx 使用 useAuth() hook 检查认证状态， 未认证时通过 useRouter().push('/login') 重定向到登录页。 AuthProvider 的 isLoading 状态防止闪烁（骨架屏过渡）。
